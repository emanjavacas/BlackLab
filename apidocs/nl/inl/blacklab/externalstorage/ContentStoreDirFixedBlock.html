Class ContentStoreDirFixedBlock

  • java.lang.Object
    • nl.inl.blacklab.externalstorage.ContentStore
      • nl.inl.blacklab.externalstorage.ContentStoreDirAbstract
        • nl.inl.blacklab.externalstorage.ContentStoreDirFixedBlock

  • public class ContentStoreDirFixedBlock
extends ContentStoreDirAbstract
    Store string content by id in a compound file and a TOC file. Quickly retrieve (parts of) the string content. Stores files in a file containing fixed-length (4K) blocks of zipped UTF-8. A file allocation table keeps track of each file's blocks as well as the character offset associated with each block so we can quickly access the data. Free blocks will be re-used to save space.

        public String[] retrieveParts(int contentId,
                     int[] start,
                     int[] end)
        Retrieve one or more substrings from the specified content. This is more efficient than retrieving the whole content, or retrieving parts in separate calls, because the file is only opened once and random access is used to read only the required parts. NOTE: if offset and length are both -1, retrieves the whole content. This is used by the retrieve(id) method.
